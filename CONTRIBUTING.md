# Contributing

Thanks for your interest in contributing to Smart Drink Fridge.

Bug reports, feature ideas and pull requests are welcome.

## Reporting bugs

Before opening a new issue, please check if a similar issue already exists.

When reporting a bug, please include:

- What you expected to happen
- What actually happened
- Raspberry Pi or system model
- Operating system
- Docker or native installation
- Relevant logs or error messages
- Steps to reproduce the issue

Please do not include passwords, API tokens, private keys or other sensitive information.

## Feature requests

Feature ideas are welcome.

Please describe:

- What you would like to add
- Why it would be useful
- How you imagine it working

## Pull requests

If you want to contribute code:

1. Fork the repository
2. Create a new branch for your changes
3. Make and test your changes
4. Open a pull request

Please keep pull requests focused on one change or feature where possible.

For larger changes, opening an issue first is recommended so the idea can be discussed before implementation.

## Testing

Please test your changes before submitting a pull request.

For Docker-related changes, make sure the containers still build and start correctly.
